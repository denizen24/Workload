# UI Checklist (Перед merge)

Чеклист для быстрой проверки новых UI-изменений.

## A. Консистентность

- [ ] Используются классы контракта (`ui-btn`, `ui-input`, `ui-card`, т.д.).
- [ ] Нет повторяющихся ad-hoc классов для одинаковых ролей.
- [ ] В секции есть не более одного primary action.

## B. Состояния

- [ ] У кнопок есть loading/disabled поведение.
- [ ] Для асинхронных списков есть skeleton или явный loading-block.
- [ ] Для “пусто” есть `ui-empty-state`.
- [ ] Для ошибок есть контрастный текст (не бледные оттенки в light theme).

## C. Accessibility

- [ ] Интерактивы доступны с клавиатуры.
- [ ] Есть `focus-visible` ring на кнопках/инпутах.
- [ ] Контраст вторичного текста читаем на dark/light теме.
- [ ] Маленькие action-кнопки имеют достаточную hit area.

## D. Визуальная иерархия

- [ ] Заголовки/описания/подписи следуют scale (`title`, `ui-muted`, `ui-text-caption`).
- [ ] Карточки используют `ui-card` и не смешивают уровни поверхности хаотично.
- [ ] Таблицы используют `ui-table-head`/`ui-table-row`.

## E. Theme / Tokenization

- [ ] Нет новых inline hex-цветов в JSX без причины.
- [ ] Для сложных областей (chart/board) используется модуль палитры из `src/theme`.
- [ ] При добавлении новых цветовых ролей обновлены токены в `tailwind.config.ts`.
